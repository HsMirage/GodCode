开发一个名为CodeAll的独立多agent协同（并且不同agent由不同LLM模型驱动）的方式工作软件，该软件需深度融合项目内 @参考项目 文件夹中多个应用的核心优势，打造功能全面、架构先进的AI辅助开发平台。具体开发要求如下：

### 技术栈选择
建议采用"pnpm + Electron + React + TypeScript + Claude Code SDK"作为核心技术栈，开发团队可根据项目实际需求与技术兼容性进行适当调整，但需确保技术选型的先进性、稳定性和可维护性。

### 架构设计要求
1. 集成oh-my-opencode的多多agent由不同LLM智能体推动的协同编程框架，实现以下核心架构特性：
   - 多agent（不同agent由不同LLM模型驱动）协同工作机制
   - 并行后台任务处理能力
   - 完善的Hook生命周期治理系统
   - 任务持续执行与自动续跑功能
2. 在omo的底层基础上融合eigent的多Agent协同架构，实现：
   - 复杂工作流程的智能子任务拆解
   - 子任务并行执行与资源优化分配
   - 任务依赖管理与执行顺序控制

### 功能实现规范
1. 完整移植openclaw的核心能力模块，确保其功能在新软件中实现无缝集成与正常运行
2. 实现hello-halo的内嵌浏览器功能及AI自动操控浏览器技术：
   - 浏览器内核集成与窗口管理
   - AI驱动的自动化浏览与操作能力
   - 网页内容分析与信息提取功能
3. 开发多视图并行的"工作台"式布局：
   - 支持多任务窗口并行显示
   - Agent工作产物的实时可视化
   - 任务执行过程的全程追踪与状态展示
4. 实现自定义API模型接入系统：
   - 兼容OpenAI-compatible API规范
   - 模型配置与管理界面
   - API密钥安全存储与使用机制

### 界面设计标准
1. 整体UI风格采用hello-halo+eigent的设计语言，实现：
   - 现代化、简洁直观的用户界面
   - 多视图并行的工作台布局
   - 响应式设计，适配不同屏幕尺寸
2. 借鉴eigent的前端页面设计布局，优化：
   - 用户交互流程
   - 功能模块组织方式
   - 信息层级与视觉引导
3. 开发关键UI组件：
   - Agent产物可视化组件，支持文本、代码、图像等多种格式展示
   - 任务追踪组件，实时显示Agent工作状态与进度
   - 后台任务查看面板，支持任务管理与操作
4. 实现完善的设置界面：
   - LLM模型可视化管理（添加、编辑、删除）
   - Agent配置与参数调整
   - LLM模型与Agent的绑定关系管理界面

### 技术实现要求
1. 代码复用与整合：
   - 允许直接复制 @参考项目 中各文件夹内的代码
   - 确保复用代码的兼容性与可维护性
   - 对复用代码进行必要的适配与重构，确保符合新项目架构
2. 独立性要求：
   - 软件必须作为独立应用运行，不依赖原有项目环境
   - 完整的依赖管理与打包机制
   - 自包含的运行环境配置
3. 系统集成与性能：
   - 实现各模块间的无缝集成与通信
   - 保证整体系统的稳定性与响应性能
   - 优化资源占用，避免内存泄漏与性能瓶颈

### 开发流程规范
1. 第一阶段：核心架构设计
   - 完成系统总体架构设计文档
   - 确定模块划分与接口定义
   - 设计模块间通信机制与数据流转方案
   - 制定代码规范与项目结构
2. 第二阶段：功能模块集成
   - 依次集成各参考项目的核心功能模块
   - 实现模块间的基础通信
   - 开发核心功能的单元测试
3. 第三阶段：联调与测试
   - 进行模块间联调，解决集成问题
   - 执行整体功能测试
   - 进行性能优化与问题修复
4. 第四阶段：UI/UX优化
   - 完善界面设计与交互体验
   - 进行用户体验测试与优化
   - 实现最终视觉效果与动效

2. 源代码与文档：
   - 完整、可编译的源代码
   - 详细的代码注释与API文档
   - 架构设计文档与模块说明
3. 测试报告：
   - 各功能模块的单元测试报告
   - 系统集成测试报告
   - 性能测试报告，重点验证：
     * 多Agent协同工作时的稳定性

     * 长时间运行的系统资源占用情况
