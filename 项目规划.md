建议采用"pnpm + Electron + React + TypeScript"作为核心技术栈，开发团队可根据项目实际需求与技术兼容性进行适当调整，但需确保技术选型的先进性、稳定性和可维护性。

1. 集成oh-my-opencode的多agent由不同LLM智能体推动的协同编程框架，实现以下核心架构特性：
   - 多agent（不同agent由不同LLM模型驱动）协同工作机制
   - 并行后台任务处理能力
   - 完善的Hook生命周期治理系统
   - 任务持续执行与自动续跑功能
2. 在omo的底层基础上融合eigent的多Agent协同架构，实现：
   - 复杂工作流程的智能子任务拆解
   - 子任务并行执行与资源优化分配
   - 任务依赖管理与执行顺序控制

### 功能实现规范

## 多agent协同（不同Agent由不同LLM模型驱动）落地实现说明（已完成）

> 本节基于 `参考项目/oh-my-opencode` 与 `参考项目/eigent` 的协同机制，给出 CodeAll 当前可运行实现与验收口径。

### 1) 架构落地（Workforce + Delegate）

- `src/main/services/workforce/workforce-engine.ts`
  - 负责任务分解、DAG 依赖调度、并发执行（`MAX_CONCURRENT=3`）。
  - 每个子任务强制路由为 `subagent_type` 或 `category`（二选一），防止职责混乱。
  - 支持计划文件执行（`.sisyphus/plans/*.md`）与自动补审查任务（Haotian -> LeiGong）。
- `src/main/services/delegate/delegate-engine.ts`
  - 负责原子任务执行与模型解析。
  - 支持按 `Agent` 或 `Category` 分配模型，并在任务记录中写入 `assignedModel`。
  - 支持任务运行日志（Run + logs）与工具调用日志。

### 2) 多模型驱动（每个Agent/Category独立模型）

- `src/main/services/binding.service.ts`
  - Agent 绑定模型、Category 绑定模型、温度、Prompt 覆盖。
  - 绑定优先，其次系统默认模型。
- `src/main/services/llm/model-resolver.ts`
  - Fallback chain 解析：优先返回“实际可用的已配置模型”。
  - 系统默认模型无 provider 前缀时，可从已配置模型中推断 provider。
- `src/main/services/delegate/delegate-engine.ts`
  - 修复可用模型集合读取（`modelName`）。
  - 在 fallback 场景自动解析可用凭据，避免“有模型但无法执行”。

### 3) 可观测性与协同追踪（参考 Eigent 的事件化思路）

- Workflow 返回结构包含 `executions`（逻辑任务 -> 持久任务ID/执行模型/来源）。
- 工作流完成文本会显示“任务分配 + 实际模型”。
- 任务节点展示 `assignedModel`，Agent 面板可展示实时模型占用。

### 4) 前端协同工作台

- `src/renderer/src/components/workflow/WorkflowView.tsx`
  - 可视化 DAG、状态流转、依赖边。
- `src/renderer/src/store/agent.store.ts`
  - 同时纳入 Agent 与 Category 执行通道，避免类别任务“看不见”。
- `src/renderer/src/components/agents/AgentList.tsx`
  - 展示每个执行通道当前任务与模型。

### 5) 对照参考项目的吸收点

- 吸收 OMO：`Category + Subagent` 双路由、fallback chain、多角色协作、计划执行语义。
- 吸收 Eigent：以工作流事件和任务执行记录为核心的可观测协同状态。

### 6) 验收标准（可直接执行）

1. 绑定至少 2 个不同模型（例如 `haotian`、`qianliyan`、`dayu` 分别绑定不同模型）。
2. 在聊天中使用 `haotian` 发送复杂任务（触发 workforce）。
3. 在“流程图/代理”视图确认：
   - 子任务被拆分并存在依赖关系；
   - 不同任务显示不同 `assignedAgent/Category`；
   - 至少两个任务的 `assignedModel` 不同。
4. 检查任务表与运行日志：
   - `task.assignedModel` 已写入；
   - `Run` 日志存在对应执行记录。
