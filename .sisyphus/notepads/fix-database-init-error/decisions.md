# Decisions Log

## 2026-01-30 Session 1

### 重试策略

- **决策**: 最多 3 次尝试 (1 初始 + 2 重试)
- **理由**: 平衡恢复能力和性能影响
- **退避时间**: 失败后 1s, 3s (指数增长)

### 进程清理安全性

- **决策**: 仅杀死路径匹配 `@embedded-postgres` 的进程
- **理由**: 防止误杀用户系统安装的 PostgreSQL
- **实现**: 使用 wmic 获取进程路径并过滤

### 数据目录删除策略

- **决策**: 仅在 `PG_VERSION` 不存在且 initdb 失败时删除
- **理由**: 保护已有用户数据，仅清理首次初始化失败的损坏目录

### Import 路径规范

- **决策**: 使用别名 `@/main/services/` 而非相对路径
- **理由**: 与现有测试风格一致，确保 vitest mock 生效
