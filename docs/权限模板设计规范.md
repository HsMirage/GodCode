# 权限模板设计规范

本文给出 CodeAll 权限模板（Safe / Balanced / Full）的设计规范，并明确与 `auto / confirm / deny` 的映射关系。

## 1. 现状基线

当前权限模型定义在：
`/Users/mirage/AI/AiWork/CodeAll/src/main/services/tools/permission-policy.ts`

核心类型：

```ts
type ToolPermissionLevel = 'auto' | 'confirm' | 'deny'
```

系统按工具配置生成安全分级：

- `safe`：自动执行（auto）
- `moderate`：需确认（confirm）
- `dangerous`：危险工具（dangerous=true）

辅助函数：`getToolSafetyLevel`、`groupToolsBySafety`。

## 2. 模板目标

通过模板为用户提供一键策略，不要求逐个工具手动配置。

- **Safe**：默认最保守，优先 deny/confirm
- **Balanced**：读多写少，常见开发动作可用
- **Full**：高自主度，除显式禁用外尽量放开

## 3. 模板映射建议

以下是推荐映射策略（作为规范，不等于已全部实现 UI）：

### 3.1 Safe

- 文件读取/搜索：`auto`
- 文件写入、浏览器交互、bash：`confirm` 或 `deny`
- 未知工具：`deny`

适合：审查模式、高安全环境。

### 3.2 Balanced

- 读取/搜索：`auto`
- 常规写入：`confirm`
- bash/高风险工具：`confirm`
- 未知工具：`confirm`

适合：默认研发场景。

### 3.3 Full

- 常见工具：`auto`
- 危险工具：至少 `confirm`（不建议完全 auto）
- 未知工具：`confirm`

适合：受信任环境下的高效率执行。

## 4. 关键安全约束

1. alias 归一化后再做权限判断，禁止绕过。
2. 高风险工具不得在无可见提示下静默执行。
3. 未知工具不得默认 auto。
4. 模板切换必须可审计（记录变更前后差异）。

## 5. 落地接口建议

- 在设置层保存当前模板 ID（global/space scope）。
- 模板展开后写入 `PermissionPolicy` 自定义覆盖。
- 通过 `setting:schema-list` 暴露模板配置项定义能力。

相关设置体系：
`/Users/mirage/AI/AiWork/CodeAll/src/main/services/settings/schema-registry.ts`

## 6. 验收标准

1. 模板切换后，同一工具行为稳定可复现。
2. 高风险命令不会在 Safe/Balanced 中静默执行。
3. alias 命中前后权限结果一致。
4. 审计日志能还原模板与工具权限决策。
