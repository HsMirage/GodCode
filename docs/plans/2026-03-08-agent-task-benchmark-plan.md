# Agent 任务能力基准集方案（2026-03-08）

> 对应任务：P3-1 建立任务能力基准集

## 目标

- 建立一套固定、可重复运行的任务基准集，用来持续评估 Agent 的真实任务完成能力。
- 让后续每周或每版本都能基于同一批任务对比优化前后效果，而不是依赖零散主观判断。
- 为 `P3-2` 任务级 KPI 仪表盘提供统一输入样本与维度口径。

## 基准集维度

| 维度 | 说明 | 代表基准任务 |
|---|---|---|
| 简单代码任务 | 单点实现或修复，边界清晰、验收简单 | `B01-simple-bug-fix` |
| 跨模块实现任务 | 涉及主进程 / renderer / shared 等多模块协作 | `B02-cross-module-feature` |
| 只读分析任务 | 仅允许检索、阅读、总结，不允许改码 | `B03-readonly-architecture-review` |
| 高风险审批任务 | 必须命中审批闸门并验证批准/拒绝行为 | `B04-sensitive-config-change` |
| 浏览器自动化任务 | 依赖浏览器导航、提取、快照等工具 | `B05-browser-data-extraction` |
| 恢复/续跑任务 | 验证刷新/崩溃后的上下文恢复与续跑 | `B06-recovery-after-refresh` |
| 发布验收任务 | 验证构建/门禁/证据归档的发版闭环 | `B07-release-preflight` |

## 每个基准任务的固定记录字段

- 输入描述
- 期望路由
- 允许工具
- 期望输出
- 是否需要审批
- 是否通过
- 失败原因

## 代码落点

- 基准集定义：`src/shared/task-benchmark-suite.ts`
- 聚合与汇总 helper：`src/shared/task-benchmark-suite.ts`
- 单测：`tests/unit/shared/task-benchmark-suite.test.ts`

## 运行建议

- 每周至少运行 1 次完整基准集。
- 每个版本发布前至少运行 1 次完整基准集。
- 对高风险审批、恢复续跑、发布验收三类任务，保留单独的失败原因归档。
- 后续可将运行结果接入仪表盘或版本报告，统一展示通过率、审批覆盖率与缺失样本。

## 当前范围说明

- 本轮先完成“固定任务集 + 汇总 helper + 文档口径”三件事。
- 暂不在本轮实现自动调度器或可视化面板；这些结果可由 `P3-2` 继续消费。
