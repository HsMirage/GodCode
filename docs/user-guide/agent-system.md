CodeAll 使用多智能体系统来处理不同类型的任务。每个 Agent (智能体) 都有特定的能力和工具集，适合处理不同类型的工作。

## Agent 概述

CodeAll 的 Agent 系统基于中国神话/历史人物命名，分为两大类:

1. **主要智能体 (Primary Agents)**: 能够执行写入操作，可以修改文件、执行命令
2. **辅助智能体 (Subagents)**: 只读操作，专注于分析、搜索、咨询

为了兼容性，每个 Agent 都有英文别名:

## 主要智能体

### 伏羲 (FuXi) - 战略规划器

**别名**: prometheus

**描述**: 战略规划器，采用面试模式创建详细的工作计划。

**能力**:
- 通过系统化问答澄清需求
- 创建结构化的实施计划
- 预判潜在风险和障碍

**可用工具**:
- `read` - 读取文件
- `write` - 写入文件
- `edit` - 编辑文件
- `bash` - 执行命令
- `webfetch` - 获取网页内容

**适用场景**:
- 新项目启动规划
- 复杂功能设计
- 重构方案制定

**使用示例**:
```
用户: 我想给项目添加用户认证功能

伏羲: 好的，我来帮你规划用户认证功能的实现。让我先问几个问题:
      1. 你希望支持哪些认证方式？(邮箱/密码, OAuth, etc.)
      2. 需要支持哪些 OAuth 提供商？
      3. 是否需要双因素认证？
      ...
```

### 昊天 (HaoTian) - 主编排器

**别名**: sisyphus

**描述**: 主编排器，负责任务分解、并行委派和 TODO 工作流管理。

**能力**:
- 将复杂任务分解为子任务
- 并行委派任务给其他 Agent
- 管理任务依赖和进度

**可用工具**:
- `read` - 读取文件
- `write` - 写入文件
- `edit` - 编辑文件
- `bash` - 执行命令
- `glob` - 文件模式匹配
- `grep` - 内容搜索
- `delegate_task` - 委派任务

**适用场景**:
- 需要多步骤完成的复杂任务
- 需要并行处理的工作
- 需要协调多个 Agent 的场景

**使用示例**:
```
用户: 重构整个项目的错误处理机制

昊天: 这是一个复杂任务，我来分解执行:

      任务分解:
      1. [千里眼] 分析当前错误处理模式
      2. [白泽] 设计新的错误处理架构
      3. [昊天] 实施重构
      4. [白泽] 代码审查

      正在并行执行任务 1 和 2...
```

## Agent 选择指南

根据任务类型选择合适的 Agent:

```
任务类型                    推荐 Agent
────────────────────────────────────────
新功能开发 (计划)          → 伏羲 (fuxi)
新功能开发 (执行)          → 鲁班 (luban)
复杂任务协调               → 昊天 (haotian)
代码搜索/探索              → 千里眼 (qianliyan)

```

## Agent 协作机制

CodeAll 的 Agent 可以相互协作:

### 委派 (Delegation)

主要 Agent 可以将子任务委派给辅助 Agent:

```
用户 → 昊天(主编排器)
           ├── 委派给 千里眼: 搜索代码
           ├── 委派给 白泽: 审查设计
           └── 自己执行: 实施修改
```

### 工作流示例

**复杂功能开发流程**:

1. **伏羲** 创建工作计划
2. **雷公** 审查计划完整性
3. **夸父** 按计划分发任务
4. **千里眼** 探索相关代码
5. **鲁班** 实施开发
6. **白泽** 代码审查
7. **昊天** 协调并合并

---

## 最佳实践

1. **选择合适的 Agent**: 简单任务用轻量 Agent，复杂任务用全能 Agent
2. **利用委派机制**: 让主 Agent 自动选择合适的辅助 Agent
3. **任务分解**: 大任务拆分成小任务，可以并行处理
4. **阅读 Agent 输出**: Agent 会报告其使用的工具和推理过程
